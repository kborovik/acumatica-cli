# SPEC ‚Äî acu CLI (acumatica-cli)

## ¬ßG GOAL

Configure Acumatica ERP purely from source ‚Äî no UI, no Configuration Wizard. Idempotent ops: `apply` (keyed upserts), `diff` (drift detect, exit 2 on drift); `tenant create` bootstraps at birth. Done = clone data repo, decrypt, `acu tenant create` ‚Üí `acu apply` ‚Üí `acu diff` ‚Üí byte-identical configured tenant every time.

## ¬ßC CONSTRAINTS

- Repo = `acu` CLI only. Data (`baseline/`, `.env.gpg`) lives in separate data repos (sibling `acumatica-devops`); infra = sibling `acumatica-infra`; blog = sibling `acumatica-blog`. C# customization projects out of scope (exclusion: bootstrap package ships one C# CustomizationPlugin package ‚Äî ¬ßT.11; endpoint item returns when ¬ßT.12 lands).
- Python ‚â• 3.12; click, httpx, pydantic, pydantic-settings, rich, pyyaml; uv build; module `acumatica_cli`; entry `acu`.
- Default suite fully offline: SSH = monkeypatched `subprocess.run`, REST = `httpx.MockTransport`; no live instance needed; `make check` gate offline-only. Opt-in live tier: pytest marker `e2e`, deselected via addopts, runs only via `make e2e`; e2e self-contained ‚Äî single-org synthetic company scaffolded from packaged `config init` templates into tmp dirs, no repo-root data symlinks, no dataset tenants (`SalesDemo|T100|U100` stay CLI surface, never test fixtures); multi-org out of demo scope (paid engagement).
- Every cmd except `--dry-run` talks to live instance. Final verification live vs acu-dev1 (`acu-dev1.vm.internal`; needs tailnet + GPG key): `cd ~/github/acumatica-devops && make decrypt && make diff` or `make e2e` here (self-contained per the e2e constraint; `.env`/`.env.gpg` real files @ repo root).
- Unconfigured tenant fails most entities (500 `PXSetupNotEnteredException` or 403 feature-gated) until bootstrap (features + company/branch); built-in endpoints can't bootstrap; `PUT CompaniesStructure` dead; payment terms have no Default-endpoint entity; financial currency (CM202000) has none either ‚Äî REST `Currency` = CM201000 currency list only.
- `ac.exe` has no snapshot support ‚Äî reference data as code is the primary route.
- Default `config init` = finance-minimal (offline e2e stays green). `--flavor distribution` = opt-in full demo seed (lab5 Demo Tenant Factory class). Distribution non-goals: multi-org, multicurrency, full tax engine; not a production data-repo cutover path.
