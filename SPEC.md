# SPEC ΓÇö acu CLI (acumatica-cli)

## ┬ºG GOAL

Configure Acumatica ERP purely from source ΓÇö no UI, no Configuration Wizard. Idempotent ops: `apply` (keyed upserts), `diff` (drift detect, exit 2 on drift); `tenant create` bootstraps at birth. Done = clone data repo, decrypt, `acu tenant create` ΓåÆ `acu apply` ΓåÆ `acu diff` ΓåÆ byte-identical configured tenant every time.

## ┬ºC CONSTRAINTS

- Repo = `acu` CLI only. Data (`baseline/`, `.env.gpg`) lives in separate data repos (sibling `acumatica-salesdemo`); infra = sibling `acumatica-infra`; blog = sibling `acumatica-blog`. C# customization projects out of scope (exclusion: bootstrap package ships one C# CustomizationPlugin package ΓÇö ┬ºT.11; endpoint item returns when ┬ºT.12 lands).
- Python ΓëÑ 3.12; click, httpx, pydantic, pydantic-settings, rich, pyyaml; uv build; module `acumatica_cli`; entry `acu`.
- Default suite fully offline: SSH = monkeypatched `subprocess.run`, REST = `httpx.MockTransport`; no live instance needed; `make check` gate offline-only. Opt-in live tier: pytest marker `e2e`, deselected via addopts, runs only via `make e2e`.
- Every cmd except `--dry-run` talks to live instance. Final verification live vs acu-dev1 (`acu-dev1.vm.internal`; needs tailnet + GPG key): `cd ~/github/acumatica-salesdemo && make decrypt && make diff` or `make e2e` here (data symlinks `baseline`/`bootstrap`/`setup` ΓåÆ sibling data repo; `.env`/`.env.gpg` real files @ repo root).
- Unconfigured tenant fails most entities (500 `PXSetupNotEnteredException` or 403 feature-gated) until bootstrap (features + company/branch); built-in endpoints can't bootstrap; `PUT CompaniesStructure` dead; payment terms have no Default-endpoint entity; financial currency (CM202000) has none either ΓÇö REST `Currency` = CM201000 currency list only.
- `ac.exe` has no snapshot support ΓÇö reference data as code is the primary route.

- cmd: `acu tenant list|create|delete` ΓåÆ tenant CRUD over SSH; create: `--id` ! + `--login` ! + `--type`/`--parent`/`--hidden`/`--no-init` ?, then chains bootstrap publish (data plane, content-digest skip per V4) ΓÇö tenant create + bootstrap one step; --type validates client-side: choice `SalesDemo|T100|U100` (V12-verified dataset folders on box; `System` excluded ΓÇö system-tenant dataset), omitted ΓåÆ clean tenant, non-member ΓåÆ error naming allowed set before any SSH, help text lists exact names; create resumable: login already exists ΓåÆ skip ac.exe create (`--id` ! match existing CompanyID, else error), init + publish chain still runs ΓÇö republish route for existing tenants; delete: `--id` + confirm prompt, `--yes` skips

- cmd: `acu diff [files|dirs]` ΓåÆ GET by `$filter` on key fields, compare normalized; args ? ΓåÆ same default set as `apply`; optimization-500 (delegate-view fields) ΓåÆ retry record via key-URL GET `/<entity>/<key1>[/<key2>...]` (YAML `key` order); drift ΓåÆ exit 2
- cmd: `acu extract [--out <dir>] [--only <name>]... [--force] [--dry-run]` ΓåÆ inverse of apply: GET live tenant state ΓåÆ seed YAML file set (`bootstrap/` `baseline/` `setup/`); manifest-driven ΓÇö package data `extract_manifest.yaml` per-entity: endpoint (symbolic `bootstrap` resolves `seed.BOOTSTRAP_ENDPOINT` @ load), keys, file (numbered prefix = V22 order), strip deny-list or include allow-list (mutually exclusive), filter ? - cmd: `acu schema [--out <dir>]` ΓåÆ OpenAPI dump ΓåÆ `schemas/` (gitignored ~3 MB; regenerate, never version)
- cmd: `acu config show` ΓåÆ emit resolved `.env`-format doc: header comment + `ACU_*` keys w/ resolved values (defaults merged, flag overrides reflected); `ACU_PASSWORD` never emitted any form ΓåÆ output round-trips (`acu config show > .env` reloads identical, password excluded); ! same `Instance` resolution path as live cmds ΓÇö no parallel resolution
- cmd: `acu config init [--host <host>] [<dir>]` ΓåÆ scaffold data repo: `.env` all `ACU_*` keys as placeholders (`--host` substitutes host into `ACU_BASE_URL`/`ACU_SSH` values, placeholder `erp.example.com`; `ACU_USER=admin` + `ACU_PASSWORD=` blank, never real secrets; header comment ΓåÆ `acu config show`), `baseline/` minimal generic company (`10-subaccounts` ZERO segment + `20-accounts` ~10 generic + `40-ledger` ACTUAL + `50-gl-preferences` + `60-ledger-company` org-ledger link + `90-uoms`; numbered prefixes teach V22 order), `bootstrap/company.yaml` + `bootstrap/credit-terms.yaml` (per-file `endpoint:` override example) + `bootstrap/features.yaml` (six + SubAccount ΓÇö template-set feature closure), `setup/` GL action chain (`10-financial-year` + `20-master-calendar` + `30-open-periods`; `OrganizationID` COMPANY = company template `AcctCD`, year values user-edited like `host` placeholder), `.gitignore` (`.env`, `schemas/`); `<dir>` ? cwd, created if absent; per-file skip-if-exists ΓåÆ `skip <file> (exists)`, exit 0, never overwrites; templates = package data via `importlib.resources`; no `git init`, no gpg
- cmd: `acu config check` ΓåÆ read-only preflight, dependency order, one `ok`/`fail` line each: discovery (walk-up finds `.env` + parse + `ACU_BASE_URL`) ΓåÆ secrets (`ACU_PASSWORD` resolved) ΓåÆ REST (login + landed-tenant verify + logout) ΓêÑ ssh (trivial remote cmd via `_ssh`); REST + ssh probe independently, discovery/secrets fail stops; no PUTs, no tenant CRUD; exit 0 all pass, 1 any fail (2 stays drift)
- cfg: `Instance` = pydantic-settings `BaseSettings`, env prefix `ACU_`, all keys ?: `ACU_BASE_URL` (REST root: scheme + host + site path), `ACU_SSH` (control plane, full `user@host`), `ACU_TENANT`, `ACU_API_VERSION` (default `25.200.001`; endpoint name half hardcoded `Default` ΓÇö custom endpoints = per-call seed `endpoint:` overrides), `ACU_USER` (default `admin`), `ACU_PASSWORD`; flags override env per key; install-layout values = `config.py` constants (`AcumaticaERP` ΓÇö IIS pool coupling documented @ `recycle_app_pool` call site, `C:\Acumatica\AcumaticaERP`, `ac.exe` path, `AcumaticaDB`), not config surface
- env: `.env` ? ΓÇö sole config file: where + secrets as `ACU_*` vars; discovery sentinel, walk-up per V3; encrypted at rest as `.env.gpg`; absent ΓåÆ resolution runs on flags + process environment alone; password ! resolve via `--password` or `ACU_PASSWORD`; username ? via `--username` or `ACU_USER`, default `admin`
- data: `baseline/*.yaml` ΓåÆ `entity` / `key` (string or list) / `records` + `endpoint` ? (per-file contract-endpoint override, e.g. `Bootstrap/1.4.0`); parsed by `seed.py`
- data: `setup/*.yaml` ΓåÆ action file: `action` / `entity` / `record` + `parameters` ? + `endpoint` ? + `done_when` (`{entity?, endpoint?, filter?}`; entity+endpoint default to action's); one `record` per file ΓÇö multi-invocation = multiple numbered files; realized `POST /entity/<endpoint>/<Entity>/<Action>` body `{entity, parameters}` ΓåÆ 204 done or 202 poll `Location`; parsed by `seed.py` dispatch on `action:` key ΓÇö `apply`/`diff` absorb kind transparently
- data: `bootstrap/features.yaml` ΓåÆ ? list of `FeaturesSet` property names (DAC bit flags) ΓåÆ plugin `Enabled` set @ package build; absent ΓåÆ built-in six (FinancialModule, FinancialStandard, DistributionModule, Inventory, Branch, MultiCompany)
- api: `/entity/Default/<api_version>/` (default `25.200.001`) ΓåÆ cookie-session httpx; values wrapped `{"Field": {"value": ...}}`; PUT = keyed upsert
- ssh: `ac.exe -cm:CompanyConfig` + `sqlcmd` over `ssh` ΓåÆ remote shell PowerShell; `exit $LASTEXITCODE` appended so failures propagate
- pkg: pypi.org `acumatica-cli` ΓåÆ `pip install acumatica-cli`; publish = GitHub Actions `release.yml` on `release: published`, trusted publishing (OIDC), `uv build` sdist+wheel; no stored API token

## ┬ºV INVARIANTS

V1: two-plane split ΓÇö control plane = SSH (`tenant.py`, tenant CRUD only); data plane = REST (`client.py`); never mixed
V2: two source kinds never mixed ΓÇö `baseline/*.yaml` = what, `.env` = where + secrets (`ACU_*` vars, never what); both live in data repos, not here; package-embedded config = what ΓÇö bootstrap feature set sources from data-repo `bootstrap/features.yaml`, never hardcoded in plugin source
V3: discovery ΓÇö walk up from cwd to first dir containing `.env`; found ΓåÆ loads; absent ΓåÆ resolution proceeds on globals + process environment; hard error only when required value (`ACU_BASE_URL`, `ACU_SSH`, `ACU_PASSWORD`) unresolved post-merge, error names missing key(s)
V4: idempotence ΓÇö `PUT` keyed upsert is the primitive; `diff` treats source as authoritative, extra live records not flagged; drift ΓåÆ exit 2; resume/skip gate ! verify desired state, never a marker; full audit recipe ΓåÆ `.spec/check-extras.md` ┬ºV.4
V5: tenant-map ΓÇö tenant create ! `AcumaticaERP` app-pool recycle after `ac.exe`; always send explicit valid `tenant`; data-plane session ! post-login landed-tenant verify, refuse on mismatch; symptom recipe ΓåÆ `.spec/check-extras.md` ┬ºV.5
V6: `AcumaticaClient` ! context manager ΓÇö logout even on failure (sessions count vs license API-user cap); logout ! `Content-Length: 0` (else IIS 411)
V7: `CompanyConfig` ! `-h` beside `-iname` + `-dbnew:"False"`; delete uses `Deleted` sub-key + full spec (`ParentID` + `CompanyType`); create presets admin via `-aun`/`-aup`/`-auc` ΓÇö contract API can't clear `PasswordChangeOnNextLogin`, `Login.aspx` screen flow = fallback only
V9: output ΓÇö everything through `output.py`, no bare `print()`; stdout = data, stderr = process; ASCII-only every path; exit 0 ok, 1 error, 2 drift; no `--json` ΓÇö plain text = machine interface; full audit recipe ΓåÆ `.spec/check-extras.md` ┬ºV.9

V11: REST targets versioned path only (`Default/25.200.001`), never unversioned alias
V12: `docs/ac-exe.md` + `docs/rest-api.md` verified vs live 26.101.0225 ΓÇö trust over training data, re-verify on upgrade; dumped schema (`acu schema`) = authoritative field reference

V15: cmd grammar ΓÇö exactly two forms: `acu [globals] <noun> <verb> [options]` = resource ops; `acu [globals] <verb> [options] [args]` = data plane; no third form; surface encodes V1 split; verb map ΓåÆ `.spec/check-extras.md` ┬ºV.15
