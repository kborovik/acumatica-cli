# SPEC ‚Äî acu CLI (acumatica-cli)

## ¬ßG GOAL

Configure Acumatica ERP purely from source ‚Äî no UI, no Configuration Wizard. Idempotent ops: `apply` (keyed upserts), `diff` (drift detect, exit 2 on drift); `tenant create` bootstraps at birth. Done = clone data repo, decrypt, `acu tenant create` ‚Üí `acu apply` ‚Üí `acu diff` ‚Üí byte-identical configured tenant every time.

## ¬ßC CONSTRAINTS

- Repo = `acu` CLI only. Data (`baseline/`, `.env.gpg`) lives in separate data repos (sibling `acumatica-salesdemo`); infra = sibling `acumatica-infra`; blog = sibling `acumatica-blog`. C# customization projects out of scope (exclusion: bootstrap package ships one C# CustomizationPlugin package ‚Äî ¬ßT.11; endpoint item returns when ¬ßT.12 lands).
- Python ‚â• 3.12; click, httpx, pydantic, pydantic-settings, rich, pyyaml; uv build; module `acumatica_cli`; entry `acu`.
- Default suite fully offline: SSH = monkeypatched `subprocess.run`, REST = `httpx.MockTransport`; no live instance needed; `make check` gate offline-only. Opt-in live tier: pytest marker `e2e`, deselected via addopts, runs only via `make e2e`.
- Every cmd except `--dry-run` talks to live instance. Final verification live vs acu-dev1 (`acu-dev1.vm.internal`; needs tailnet + GPG key): `cd ~/github/acumatica-salesdemo && make decrypt && make diff` or `make e2e` here (data symlinks `baseline`/`bootstrap`/`setup` ‚Üí sibling data repo; `.env`/`.env.gpg` real files @ repo root).
- Unconfigured tenant fails most entities (500 `PXSetupNotEnteredException` or 403 feature-gated) until bootstrap (features + company/branch); built-in endpoints can't bootstrap; `PUT CompaniesStructure` dead; payment terms have no Default-endpoint entity; financial currency (CM202000) has none either ‚Äî REST `Currency` = CM201000 currency list only.
- `ac.exe` has no snapshot support ‚Äî reference data as code is the primary route.

- cmd: `acu tenant list|create|delete` ‚Üí tenant CRUD over SSH; create: `--id` ! + `--login` ! + `--type`/`--parent`/`--hidden`/`--no-init` ?, then chains bootstrap publish (data plane, content-digest skip per V4) ‚Äî tenant create + bootstrap one step; --type validates client-side: choice `SalesDemo|T100|U100` (V12-verified dataset folders on box; `System` excluded ‚Äî system-tenant dataset), omitted ‚Üí clean tenant, non-member ‚Üí error naming allowed set before any SSH, help text lists exact names; create resumable: login already exists ‚Üí skip ac.exe create (`--id` ! match existing CompanyID, else error), init + publish chain still runs ‚Äî republish route for existing tenants; delete: `--id` + confirm prompt, `--yes` skips
- cmd: `acu apply [--dry-run] [files|dirs]` ‚Üí PUT each record; dir arg expands `*.yaml`; args ? ‚Üí default = existing init-scaffolded dirs @ data-repo root, fixed order (`bootstrap/`, `baseline/`, `setup/`); none exist ‚Üí error exit 1 naming dirs; dry-run lines `would PUT ‚Ä¶`, summary suffixed `(dry run)`
- cmd: `acu diff [files|dirs]` ‚Üí GET by `$filter` on key fields, compare normalized; args ? ‚Üí same default set as `apply`; optimization-500 (delegate-view fields) ‚Üí retry record via key-URL GET `/<entity>/<key1>[/<key2>...]` (YAML `key` order); drift ‚Üí exit 2
- cmd: `acu run [--dry-run] [files|dirs]` ‚Üí execute transaction scenario YAML; dir arg expands `*.yaml`; steps sequential: `put` (recursive wrap, detail lists native) | `action` (invoke + 202 poll) | `wait` (poll record until field match, timeout ‚Üí fail); `capture:` lifts server-assigned keys ‚Üí `${var}` interpolation in later steps; `expect:` `get` (key-URL field assertions) + `inquire` (contract inquiry PUT `$expand=Results`) delta assertions ‚Äî pre-run snapshot, post-run re-probe, delta compare (warm-tenant re-runnable); dry-run parses + lists steps, no HTTP; exit 0 expectations hold, 1 step error or expectation miss (2 stays drift)
- cmd: `acu extract [--out <dir>] [--only <name>]... - cmd: `acu schema [--out <dir>]` ‚Üí OpenAPI dump ‚Üí `schemas/` (gitignored ~3 MB; regenerate, never version)
- cmd: `acu config show` ‚Üí emit resolved `.env`-format doc: header comment + `ACU_*` keys w/ resolved values (defaults merged, flag overrides reflected); `ACU_PASSWORD` never emitted any form ‚Üí output round-trips (`acu config show > .env` reloads identical, password excluded); ! same `Instance` resolution path as live cmds ‚Äî no parallel resolution

- cmd: `acu config check` ‚Üí read-only preflight, dependency order, one `ok`/`fail` line each: discovery (walk-up finds `.env` + parse + `ACU_BASE_URL`) ‚Üí secrets (`ACU_PASSWORD` resolved) ‚Üí REST (login + landed-tenant verify + logout) ‚à• ssh (trivial remote cmd via `_ssh`); REST + ssh probe independently, discovery/secrets fail stops; no PUTs, no tenant CRUD; exit 0 all pass, 1 any fail (2 stays drift)
- cfg: `Instance` = pydantic-settings `BaseSettings`, env prefix `ACU_`, all keys ?: `ACU_BASE_URL` (REST root: scheme + host + site path), `ACU_SSH` (control plane, full `user@host`), `ACU_TENANT`, `ACU_API_VERSION` (default `25.200.001`; endpoint name half hardcoded `Default` ‚Äî custom endpoints = per-call seed `endpoint:` overrides), `ACU_USER` (default `admin`), `ACU_PASSWORD`; flags override env per key; install-layout values = `config.py` constants (`AcumaticaERP` ‚Äî IIS pool coupling documented @ `recycle_app_pool` call site, `C:\Acumatica\AcumaticaERP`, `ac.exe` path, `AcumaticaDB`), not config surface
- env: `.env` ? ‚Äî sole config file: where + secrets as `ACU_*` vars; discovery sentinel, walk-up per V3; encrypted at rest as `.env.gpg`; absent ‚Üí resolution runs on flags + process environment alone; password ! resolve via `--password` or `ACU_PASSWORD`; username ? via `--username` or `ACU_USER`, default `admin`
- data: `baseline/*.yaml` ‚Üí `entity` / `key` (string or list) / `records` + `endpoint` ? (per-file contract-endpoint override, e.g. `Bootstrap/1.4.0`); parsed by `seed.py`
- data: `setup/*.yaml` ‚Üí action file: `action` / `entity` / `record` + `parameters` ? + `endpoint` ? + `done_when` (`{entity?, endpoint?, filter?}`; entity+endpoint default to action's); one `record` per file ‚Äî multi-invocation = multiple numbered files; realized `POST /entity/<endpoint>/<Entity>/<Action>` body `{entity, parameters}` ‚Üí 204 done or 202 poll `Location`; parsed by `seed.py` dispatch on `action:` key ‚Äî `apply`/`diff` absorb kind transparently
- data: `bootstrap/features.yaml` ‚Üí ? list of `FeaturesSet` property names (DAC bit flags) ‚Üí plugin `Enabled` set @ package build; absent ‚Üí built-in six (FinancialModule, FinancialStandard, DistributionModule, Inventory, Branch, MultiCompany)
- data: `scenario/*.yaml` ‚Üí scenario file: `scenario` / `description` / `steps` (each `id` + one op key + `record` ? + `capture` ? + `wait` ?) + `expect`; transactions never keyed upserts ‚Äî `run` owns them, `apply`/`diff` never read `scenario/`; parsed by `run.py`
- api: `/entity/Default/<api_version>/` (default `25.200.001`) ‚Üí cookie-session httpx; values wrapped `{"Field": {"value": ...}}`; PUT = keyed upsert
- ssh: `ac.exe -cm:CompanyConfig` + `sqlcmd` over `ssh` ‚Üí remote shell PowerShell; `exit $LASTEXITCODE` appended so failures propagate
- pkg: pypi.org `acumatica-cli` ‚Üí `pip install acumatica-cli`; publish = GitHub Actions `release.yml` on `release: published`, trusted publishing (OIDC), `uv build` sdist+wheel; no stored API token
